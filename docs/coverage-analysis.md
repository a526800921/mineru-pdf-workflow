# 覆盖率优化分析

## 现状

demo20.pdf（20 页摩托车说明书）实测数据：

| 指标 | 值 |
|------|-----|
| 阈值 | 0.82 |
| 首次通过 | 11/20 页（55%） |
| 重跑后通过 | 11/20 页（55%） |
| 重跑 9 页全部失败 | 100% 重跑无效 |

重跑前后覆盖率完全不变（high 和 medium 产出文本相同），说明覆盖率低是**比较方式**问题，不是解析质量问题。

## 覆盖率计算

```
PDF 文本层（PyMuPDF get_text）
        ↓
    分词（中文拆单字，英文拆单词）
        ↓
 Markdown vs PDF token 交集
        ↓
 coverage = 匹配数 / PDF 总 token 数
```

## 低覆盖率的 5 类页面

### 1. 目录页（第 2–8 页，覆盖率 0.34–0.46）

PDF 文本层含大量省略号和页码：
```
前言 .............................................. 8
重要的注意事项 ............................... 10
```

分词后 `................` 这类字符被归一化，页码数字 8、10 变成 tokens，但 Markdown 里 MinerU 不保留省略号——大量 PDF token 在 MD 中不存在。

### 2. 表格页（第 13–16 页，部分覆盖率极低）

PDF 文本层里表格文字是**按行流式排列**的：
```
发动机 机油 更换 周期 5000km
```

MinerU 输出 Markdown 表格时文字被**打散到不同单元格**：
```markdown
| 项目 | 周期 |
|------|------|
| 发动机机油 | 5000km |
```

同一批字在两种格式里排列不同，token 位置对不上。

### 3. 图片标签页（第 12–13 页，覆盖率 0.12–0.71）

页面主体是图片，PDF 文本层只有少量标签文字。图片内嵌文字靠 VLM 识别，但 PDF 文本层没有这些内容——分母小但分子更小。

### 4. 纯文字页（第 9–11、17–20 页，覆盖率 0.85+）

正常段落页覆盖率高，说明比较逻辑本身对文字页有效。

### 5. 封面页（第 1 页，覆盖率 0.96）

封面文字少且匹配度高。

## 优化方案

### 方案 A：降阈值（改动最小）

阈值从 0.82 降到 0.6。

- 优点：一行改动，立即生效
- 缺点：可能漏过真正解析质量差的页
- demo20 效果：第 7、13 页仍低于 0.6，第 12 页 0.71 可通过

### 方案 B：用 content_list_v2.json 比较（改动中等）

不比较渲染后的 Markdown 文本，改为比较 `content_list_v2.json` 的结构化输出。

MinerU 的 content_list 保留了元素类型、位置信息，可以按**相同类型元素**比较：

- `paragraph` vs `paragraph`：提取文本比较
- `table` vs `table`：提取表格文字比较
- `title` vs `title`：提取标题比较
- `image` 元素：跳过（PDF 文本层没有图片内容）

- 优点：同类型比较更公平，表格页改善明显
- 缺点：需要重写 pdf-validate 的比较逻辑

### 方案 C：分类阈值（改动较大）

按页面内容类型设置不同阈值：

| 页面类型 | 阈值 | 判断方式 |
|----------|------|----------|
| 文字页（paragraph > 80% 元素） | 0.82 | content_list 类型统计 |
| 表格页（含 table 元素） | 0.40 | 同上 |
| 目录页（含大量 `......` 文本） | 0.30 | PDF 文本检测连续点号 |
| 图片页（image 元素 > 30%） | 跳过 | 同上 |

- 优点：精准，不同页不同标准
- 缺点：规则复杂，维护成本高

### 方案 D：句子级检索比较（改动最大）

不拆单字，按完整句子/段落检查：

1. 从 PDF 文本层提取句子（≥10 字）
2. 在 Markdown 中检索每个句子是否出现（模糊匹配）
3. coverage = 找到的句子数 / 总句子数

- 优点：最接近人工判断，表格页也能匹配
- 缺点：实现复杂，模糊匹配有误判风险

## 建议路径

1. **立即**：阈值降到 0.6（方案 A），解决 80% 的误报重跑
2. **短期**：改用 content_list 比较（方案 B），解决表格页
3. **长期**：句子级匹配（方案 D），接近人工标准
