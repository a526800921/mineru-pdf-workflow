# 缺陷单：MinerU 目录页过度生成——每个单页段各自吐出整本目录

- 状态：待处理（仅记录，未修复）
- 发现日期：2026-07-11
- 样本：demo20.pdf（20 页摩托车手册），目录物理页为第 2–8 页
- 归属范围：目录页解析（`coverage-validation-optimization` / `per-page-anchors` / toc 处理）
- **不属于** `table-text-omission-detection` 计划（该计划只处理 p16 表格字段遗漏）
- 相关：[toc-coverage-merge-block](toc-coverage-merge-block.md)

## 现象

合并产物 `pdf/demo20/demo20.md` 中，第 2 页目录段（`<!-- pages 2-2 -->`）末尾出现条目 `制动 130`，但该条目物理上不在 PDF 第 2 页——它属于第 4 页目录区。用户在第 2 页看到了第 4 页的内容。

## 核实证据

对目录物理页 p2–p8 逐段核实（每段的 `origin.pdf` 即 MinerU 实际输入）：

| 目录页段 | 输入 origin.pdf = 源页 | 物理页 TOC 非空行 | 段 md 行 | 段 md 含末尾条目「整车关键件扭矩表 188」 |
|---|---|---|---|---|
| p0002-0002 | ✅ 逐字节相同 | 74 | 123 | 有（物理页无） |
| p0003-0003 | ✅ | 81 | 123 | 有（物理页无） |
| p0004-0004 | ✅ | 81 | 123 | 有（物理页无） |
| p0005-0005 | ✅ | 81 | 123 | 有（物理页无） |
| p0006-0006 | ✅ | 81 | 123 | 有（物理页无） |
| p0007-0007 | ✅ | 81 | 123 | 有 |
| p0008-0008 | ✅ | 13 | 123 | 有 |

**决定性证据**：7 个物理内容各不相同的目录页，其段 md **逐字节相同**（SHA-256 均为 `cb922a8dd1c07f20...`，`diff` 零差异）。连只有 13 个非空行的第 8 页，也生成了完整的 123 条目录。

对照物理事实：

- PDF 第 2 页原生文本止于 `前制动手柄 34`，不含 `制动系统`/`停放`/`整车关键件扭矩表`。
- `制动 130`、`停放 131` 物理上位于 PDF 第 4 页（第 4 页原生文本含 `停放`）。

## 根因

**分段无误**：每个单页段的输入 `origin.pdf` 与对应源 PDF 页逐字节相同，确实每次一页。

**根源是 MinerU 解析器**：识别到目录页后，它**自动补全整本目录**，输出该页物理上不存在的条目（约 98 条来自第 3–8 页）。这是目录页上的**系统性过度生成**，对 p2–p8 每页都复现且输出完全相同，不是偶发。

下游 `toc_repair` 再把这份"每页整本"的目录按逐页锚点切分，于是 `制动 130` 落到第 2 页段尾部——这是症状，不是根因。

## 可复现基线

```bash
# 1) 证明分段正确：段输入 = 源页
python3 - <<'PY'
import fitz
src = fitz.open("pdf/demo20/demo20.pdf")
org = fitz.open("pdf/demo20/segments/p0002-0002/demo20/hybrid_auto/demo20_origin.pdf")
assert src[1].get_text() == org[0].get_text(), "段输入应等于源第2页"
assert "整车关键件扭矩表" not in src[1].get_text(), "第2页物理上无末尾条目"
print("ok: 分段正确，第2页物理不含整本目录")
PY

# 2) 证明 7 个目录页段 md 逐字节相同（同一份整本目录）
for pg in 02 03 04 05 06 07 08; do
  shasum -a256 "pdf/demo20/segments/p00${pg}-00${pg}/demo20/hybrid_auto/demo20.md" | cut -c1-16
done | sort -u   # 应只输出 1 个哈希
```

## 影响

- 合并 md 目录区含大量重复/错位条目，逐页锚点归属不可靠（症状：`制动 130` 出现在第 2 页）。
- 覆盖率验证对这些目录页普遍偏低 → p2–p8 标 `review_only`（页面类型: toc），需人工复核。
- 结构化抽取若依赖目录条目页归属，会引入错误 page_start/page_end。

## 待决策（候选方案，未实施）

1. **按 origin 物理页原生文本裁剪目录条目**：只保留该页原生文本层实际存在的条目，丢弃 MinerU 补全的跨页条目。
2. **目录页跳过 MinerU，直接用 PDF 原生文本层构建目录**：目录页文本层通常完整（本样本每页原生条目清晰），可绕过解析器过度生成。
3. **合并阶段整本目录去重**：识别多段重复的同一目录块，只保留一份并重新按原生页归属切分。
4. **维持现状 + 依赖 `review_only`**：目录页始终人工复核，不自动修（当前行为）。

选定方案前应先固定本缺陷单为 Step 0 红基线，并评估对 `coverage-validation-optimization`、`per-page-anchors` 既有契约的影响。
